# ml/ model provenance

This directory holds ML model artifacts used by the Rust app (see
`config/app_global_settings.json` and `crates/clip/`, `crates/ml/`). Nothing
here is committed to git (see `.gitignore`); everything is either downloaded
or generated by the scripts in `scripts/`. See the top-level
[README.md](../README.md#2-generate-ml-models) for setup instructions and
[docs/PRIVACY.md](../docs/PRIVACY.md) for the checksum verification policy.

This file exists so you can tell, at a glance, what's actually in use and
what's safe to delete to reclaim disk space.

## In active use

### `CLIP1/`
**In use, do not delete.** The CLIP ViT-L/14 bundle the app actually loads:
`clip_model_path`, `clip_text_model_path`, and `clip_text_tokenizer_path` in
`config/app_global_settings.json` all point inside this directory
(`vision_model_fp16.onnx`, `text_model_fp16.onnx`, `tokenizer.json`).

Provenance: a Transformers.js-style ONNX export of
[`openai/clip-vit-large-patch14`](https://huggingface.co/openai/clip-vit-large-patch14),
published at [`Xenova/clip-vit-large-patch14`](https://huggingface.co/Xenova/clip-vit-large-patch14)
on Hugging Face. Reproduced by `scripts/setup_ml_models.py`, which downloads
the fp16 vision/text encoders plus tokenizer/config files. The combined
`model_fp16.onnx` also present in the upstream repo is intentionally *not*
downloaded (unused by this app, ~800 MB).

### `nsfw_classifier.onnx`
**In use, fallback.** The `clip_nsfw_weights_path` fallback: a small MLP
head that scores a 768-dim CLIP embedding for NSFW content, from LAION's
CLIP based NSFW detector (the AutoKeras-trained head shipped as
`clip_autokeras_binary_nsfw`, see below). Generated locally by
`scripts/setup_ml_models.py` from the raw TensorFlow SavedModel. The Rust
side detects this file by input rank (`crates/clip/src/nsfw.rs`).

### `nsfw_marqo_384.onnx`
**In use, preferred.** The more accurate NSFW model:
[`Marqo/nsfw-image-detection-384`](https://huggingface.co/Marqo/nsfw-image-detection-384)
(a timm ViT-tiny), scores image pixels directly (384x384) instead of a CLIP
embedding. Generated by `scripts/setup_marqo_nsfw.py`. When both this file
and `nsfw_classifier.onnx` are present, the app's autofill prefers this one.

## Safe to delete (with caveats)

### `clip_autokeras_binary_nsfw/`, `clip_autokeras_binary_nsfw.zip`
**Obsolete once `nsfw_classifier.onnx` exists.** Raw TensorFlow
AutoKeras SavedModel plus its zip archive, the source material
`scripts/setup_ml_models.py` converts into `nsfw_classifier.onnx`. Not read
by the Rust app directly (Rust only loads ONNX). Safe to delete after
`ml/nsfw_classifier.onnx` has been generated once; keep it around only if
you expect to need to regenerate/re-audit the ONNX export later (re-fetching
it means finding the original `clip_autokeras_binary_nsfw.zip` source
again).

## Orphaned experiments (unused, safe to delete)

These were downloaded while evaluating alternative embedding models. Nothing
in `config/app_global_settings.json` or the Rust crates references them.
They are not touched by any setup script. Delete them freely to reclaim
disk space; nothing will regenerate them automatically.

### `CLIP2/`
An experiment mixing an unrelated text-embedding model
([`mixedbread-ai/mxbai-embed-large-v1`](https://huggingface.co/mixedbread-ai/mxbai-embed-large-v1),
a BERT-based sentence embedder, see `mxbai-bread-large.onnx` and the BERT
`config.json` in this folder) rather than a CLIP model. Not wired into the
app. ~1.9 GB.

### `CLIP3/`
An ONNX export of
[`laion/CLIP-ViT-L-14-DataComp.XL-s13B-b90K`](https://huggingface.co/laion/CLIP-ViT-L-14-DataComp.XL-s13B-b90K),
an alternate CLIP ViT-L/14 checkpoint trained on DataComp instead of OpenAI's
original data. Evaluated as an alternative to `CLIP1/` but not adopted. Only
ships a combined vision+text `model_fp16.onnx` (no split encoders), which
doesn't match what `crates/clip/` expects anyway. ~840 MB.

### `clip-vit-l-14.onnx`
A single-file OpenAI CLIP ViT-L/14 visual encoder, previously exported
locally via `open_clip` + `torch.onnx.export` by an older version of
`scripts/setup_ml_models.py`. Superseded by `CLIP1/vision_model_fp16.onnx`
(current config no longer points at this path). No text encoder or
tokenizer was ever produced alongside it, which is part of why it was
replaced. ~1.2 GB.
